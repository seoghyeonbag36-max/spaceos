# SpaceOS — 프로젝트 마스터 노트

## 정체성
- **한 줄 정의**: 물리적 상권의 리뷰 데이터와 공실 히스토리를 결합하여 디지털 트윈 SaaS로 플랫폼화하는 프로젝트
- **핵심 가설**: "물리적 Place의 가치가 Digital Platform으로 전이되었다"
- **최종 목표**: 18~24개월 내 M&A Exit (네이버·카카오·직방 1순위)
- **자체 평가**: 투자 가능성 92점

## 차별화 포인트
- 기존 상권 분석 시장의 문제: 사후적 지표(카드매출·유동인구)만 의존, 공실 히스토리 단절, 실행 솔루션 부재
- SpaceOS의 답: 과거 폐업 사유 AI 분석 + 3D 디지털 트윈 + PPPP 실행 솔루션

## PPPP Framework
사용자가 정의한 4기능:

### 1. Platform (기능1)
- **의미**: 물리적 상권마다 하나의 플랫폼화
- **구현**: 상권 AI 추천 엔진 (GNN 기반)
- **출력**: 페르소나·예산·업종 입력 → 적합 상권/입지 Top-K 추천
- **MVP 정확도**: Top-3 70%+

### 2. Page (기능2)
- **의미**: 공실 히트맵 + 디지털 트윈
- **구현**: Three.js + Mapbox + LSTM 공실 예측
- **출력**: 3D 상권 뷰어 + 빨강/노랑/녹색 히트맵 + 건물별 10년 업종 히스토리 + 폐업 사유 LLM 요약
- **MVP 성능**: 로딩 3초, 60fps

### 3. Posting (기능3)
- **의미**: 입점 솔루션 — **외부에서 만든 AI 창업 코파일럿 프로그램을 연동해 적용** (2026-07-18 방향 확정)
- **구현**: `services/posting.py` 어댑터 — `POSTING_COPILOT_URL` 설정 시 외부 코파일럿 호출, 미설정 시 내부 3-Tier(고급화/가성비/기능중심) 비용-효용 계산 폴백
  - 고급화: 객단가 30k, 회전 3회/일, 체류 김
  - 가성비: 객단가 12k, 회전 8회/일
  - 기능중심: 배달 전용, 객단가 18k, 임대료 80% 절감, 고객 비방문
- **출력**: 시나리오별 매출·회수기간(ROI)·리스크. 코파일럿 연동 시 외부 결과를 동일 스키마로 정규화
- **미확정**: 외부 코파일럿의 연동 형태(REST API / 패키지 / 별도 앱) — 확정 시 어댑터 TODO 해소

### 4. Program (기능4)
- **의미**: 마케팅 자동화 — 2단계 (2026-07-18 방향 확정)
  1. **가게 단위**: 네이버 지도에 노출되는 상가의 사진·정보·이미지·리뷰 데이터를 활용해 해당 가게의 온/오프라인 마케팅 광고 솔루션 자동 생성
  2. **상권 단위**: Platform에서 수집한 정보(상권분석 시계열·감성·리뷰 키워드)를 바탕으로 상권 마케팅 솔루션 생성
- **구현**: `services/marketing.py` + LLM(Claude vision — 이미지 분석 내장) / LangChain
- **데이터 채널 제약(검증됨)**: 상가 기본정보=네이버 지역검색·카카오 로컬(공식 API), 리뷰성 텍스트=네이버 블로그 검색 API(공식), **플레이스 리뷰·사진은 공식 API 없음** → PoC는 크롤러(약관 리스크 내부 검증 한정), 상용은 점주 제공 데이터(B2B 온보딩 동의) 원칙
- **출력**: 온라인(채널 믹스 + LLM 콘텐츠 + A/B 테스트) / 오프라인(행사·팝업 템플릿 + 상권 적합도 매칭)
