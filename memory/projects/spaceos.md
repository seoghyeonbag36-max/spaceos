# SpaceOS — 프로젝트 마스터 노트

## 정체성
- **한 줄 정의**: 물리적 상권의 리뷰 데이터와 공실 히스토리를 결합하여 디지털 트윈 SaaS로 플랫폼화하는 프로젝트
- **핵심 가설**: "물리적 Place의 가치가 Digital Platform으로 전이되었다"
- **최종 목표**: 18~24개월 내 M&A Exit (네이버·카카오·직방 1순위)
- **자체 평가**: 투자 가능성 92점

## 차별화 포인트
- 기존 상권 분석 시장의 문제: 사후적 지표(카드매출·유동인구)만 의존, 공실 히스토리 단절, 실행 솔루션 부재
- SpaceOS의 답: 과거 폐업 사유 AI 분석 + 3D 디지털 트윈 + PPPP 실행 솔루션

## PPPP Framework
사용자가 정의한 4기능:

### 1. Platform (기능1)
- **의미**: 물리적 상권마다 하나의 플랫폼화
- **구현**: 상권 AI 추천 엔진 (GNN 기반)
- **출력**: 페르소나·예산·업종 입력 → 적합 상권/입지 Top-K 추천
- **MVP 정확도**: Top-3 70%+

### 2. Page (기능2)
- **의미**: 공실 히트맵 + 디지털 트윈
- **구현**: Three.js + Mapbox + LSTM 공실 예측
- **출력**: 3D 상권 뷰어 + 빨강/노랑/녹색 히트맵 + 건물별 10년 업종 히스토리 + 폐업 사유 LLM 요약
- **MVP 성능**: 로딩 3초, 60fps

### 3. Posting (기능3)
- **의미**: 입점 솔루션 (고급화 vs 가성비 vs 기능중심)
  - 고급화: 객단가 30k, 회전 3회/일, 체류 김
  - 가성비: 객단가 12k, 회전 8회/일
  - 기능중심: 배달 전용, 객단가 18k, 임대료 80% 절감, 고객 비방문
- **구현**: LSTM 매출 시뮬레이터 + ROI 계산기
- **출력**: 3시나리오별 12개월 매출·회수기간·리스크 + PDF 1pager

### 4. Program (기능4)
- **의미**: 온/오프라인 마케팅 자동화
- **구현**: LangChain 프롬프트 체인 + 행사 카탈로그
- **출력**:
  - 온라인: 채널 믹스 (인스타·블로그·당근·네이버 GFA) + LLM 콘텐츠 + A/B 테스트
  - 오프라인: 축제·플리마켓·공연·팝업 50개 템플릿 + 상권 적합도 매칭

## 4단계 확장 로드맵
| Phase | 기간 | 목표 |
|-------|------|------|
| 1 MVP | 1~4개월 | 3개 상권, AI 70%, 3D 트윈 프로토타입 |
| 2 확장 | 5~8개월 | 광역시 5개, AI 85%, B2B 대시보드 |
