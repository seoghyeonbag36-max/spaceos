# 10. Platform의 재정의 — "디지털 스킨십"

> 본 문서는 SpaceOS의 Platform을 "분석 도구"가 아니라 **물리적 상권을 디지털로 재정의하는 공간**으로 규정한다. 모든 P(Platform·Page·Posting·Program) 설계의 1차 기준점이다.

## 1. 기존 정의 vs 재정의

| 구분 | (X) 좁은 정의 | (O) SpaceOS의 정의 |
|------|--------------|-------------------|
| Platform이란 | "이 상권에 어떤 업종이 들어와야/위험한가"를 분석하는 엔진 | **물리적 상권을 디지털로 재정의하는 것** |
| 핵심 행위 | 데이터 분석·예측 | 상권에 대한 **첫 인식·정보·평가가 일어나는 디지털 공간**을 운영 |

## 2. 디지털 스킨십 (Digital Skinship) 가설

상권의 *존재*는 물리적이지만, 사람들의 *이용·경험에 대한 의사결정*은 온라인에서 먼저 일어난다.

```
[고객 여정]
  ① 디지털 스킨십 (First Touch)        ② 물리적 방문            ③ 디지털 기록
  AI 추천 · 인스타그램 ·       →     실제 상권/상가 경험   →   별점 · 리뷰 · 사진 ·
  네이버 지도 · 블로그                                         해시태그로 감정/평가 기록
        │                                                              │
        └───────────────  이 순환의 입구이자 출구가 'Platform' ──────────┘
```

즉, 물리 공간이라 하더라도 고객은 **먼저 디지털 스킨십을 거친다.** 따라서 상권에 대한 정보·인식을 처음 접하고 마지막에 기록하는 **디지털 공간 = Platform**이 반드시 필요하다.

## 3. Platform이 디지털 공간으로서 하는 일

1. **상권 정체성 집약**: 채널(네이버 지도/인스타/블로그)별 노출·언급량, 주 고객층 연령·소득, 감성 지수(Sentiment Score)를 하나의 디지털 프로필로 집약.
2. **첫 스킨십 면(面) 제공**: 고객이 상권을 "처음 만나는" 화면 — 상권 카드, 분위기 키워드, 대표 이미지.
3. **기록의 회수(回收)**: 방문 후 남긴 리뷰/사진/별점을 다시 데이터로 회수해 상권 프로필을 갱신(폐루프).
4. **AI 추천 엔진의 토대**: 위 데이터가 Page(히트맵), Posting(입점 솔루션), Program(마케팅)의 입력값이 된다.

## 4. 함의 (설계 원칙)

- Platform은 **백엔드 분석 결과물이자 동시에 프런트의 첫 화면**이다. → `apps/backend`(분석) + `apps/frontend`(첫 스킨십 면) 양쪽에 존재.
- 네이버 지도/네이버페이 연동은 "부가 기능"이 아니라 **디지털 스킨십의 채널 그 자체**다. (고객의 첫 접점이 네이버 지도이므로)
- 데이터는 항상 **유입(스킨십) → 경험 → 기록 → 재유입**의 폐루프로 흐르도록 저장·갱신한다.
