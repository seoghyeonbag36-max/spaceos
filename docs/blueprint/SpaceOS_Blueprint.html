SpaceOS Blueprint — 하나의 워크트리

핵심 가설: "물리적 Place의 가치는 Digital Platform으로 전이되었다" — 물리적 상권을 디지털 트윈 SaaS로 플랫폼화한다
PoC 거점: 강남구 신사동 가로수길 목표: 18~24개월 M&A Exit (네이버·카카오·직방)
2026.07.05 · sh.pac
1 — 왜 만드는가
디지털 스킨십(Digital Skinship) — 상권의 첫 접점은 이미 온라인이다
상권의 존재는 물리적이지만, 사람들의 의사결정은 온라인에서 먼저 일어난다. 이 순환의 입구이자 출구를 운영하는 것이 SpaceOS다.

  • 문제 1. 기존 상권분석은 카드매출·유동인구 등 사후 지표뿐 → 과거 폐업 사유를 AI로 분석
  • 문제 2. 건물의 공실 히스토리가 단절되어 있다 → 데이터 결합으로 건물 단위 공실을 '만들어서' 본다
  • 문제 3. 분석만 있고 실행 솔루션이 없다 → 입점(Posting)·마케팅(Program)까지 실행을 제공
▸ 왜 가로수길인가
가두상권 공실률 41.6% (서울 최고) vs 신사역 집합상가 9.99% — 거시 통계가 숨기는 건물별 실상을 디지털 트윈이 드러낸다는 SpaceOS의 핵심 가치제안을 그대로 증명하는 무대. 공실 표본이 풍부해 추정 로직 검증(precision/recall)에 최적이고, "몰락한 상권의 재생 진단" 내러티브는 자산운용사·건물주·강남구청에 즉시 통한다.
이 루프를 4개 기능으로 구현한다
▼
2 — 무엇을 만드는가
PPPP Framework — 마케팅 4P의 디지털 전환
전개 순서 = Page → Platform → Posting → Program. 눈에 보이는 지도(Page)로 신뢰를 만들고, 지능(Platform)을 얹고, 의사결정(Posting)과 실행(Program)으로 확장한다.
STEP 1 · 지금 구축 중

Page

Product/Price → Page
이유"건물이 비었다"는 데이터는 세상에 존재하지 않는다. 결합해서 만들어야 하고, 만들어야 눈에 보인다. 보여야 신뢰가 생긴다.
구현공실률 = 1 − (활성 점포 ÷ 상가 호 수). 상가정보(분자) ⊕ 건축물대장(분모) ⊕ GIS 폴리곤 결합 → 네이버 지도 위 렌더 + Three.js 3D 트윈
데이터소상공인 상가정보 · 건축물대장 · GIS건물폴리곤 · 서울 생활인구(KT) · SGIS 인구밀도 · 부동산원 임대/공실
출력건물 상태색 🟢만실 🟡부분 🟠고공실 🔴공실의심 + 시간 슬라이더 유동인구 흐름 + 건물 10년 업종 히스토리
CLAUDE DESIGN 적용
지도 위 한국형 패턴 — 건물 클릭 → 바텀시트 상세, 공실 위험도 색 MapMarkerPin + VacancyLegend. 히트맵은 brand teal 계열, 길찾기만 네이버 그린
KPI: 로딩 <3초 · 60fps · status 정확도 70%+
STEP 2 · 지능화

Platform

Place → Platform
이유상권에 대한 첫 인식·정보·평가가 일어나는 디지털 공간을 운영하는 것 — 분석 도구가 아니라 스킨십 루프의 입구이자 출구.
구현상권 디지털 프로필(노출량·고객층·감성지수) + LSTM 공실·매출 예측 + GNN 업종 시너지/잠식 분석 기반 AI 추천
데이터서울 상권분석서비스 시계열(추정매출·개폐업률·길단위인구) · LOCALDATA 개폐업 이력(LSTM 라벨) · 리뷰 크롤링
출력페르소나·예산·업종 입력 → 적합 상권/입지 Top-K 추천 + 상권 카드(분위기 키워드·대표 이미지)
CLAUDE DESIGN 적용
고객이 상권을 "처음 만나는 화면" — 상권 카드 · 감성 프로필이 앱의 첫 스킨십 면(面). AI 추천은 brand teal로 SpaceOS 고유 기능임을 표시
KPI: Top-3 정확도 70%+ (MVP) → 85% → 90%
STEP 3 · 의사결정

Posting

Promotion → Posting
이유추천만으로는 아무도 계약하지 않는다. 비용-효용을 숫자로 보여줘야 입점 의사결정이 일어난다 — 분석을 '실행'으로 잇는 관문.
구현LSTM 매출 시뮬레이터 + ROI 계산기. 전략 3종: 고급화(객단가 3만·회전 3회) / 가성비(1.2만·8회) / 기능중심(배달·임대료 80%↓)
데이터공정위 가맹정보(가맹금·인테리어 비용·평균매출) · 상권분석 추정매출/소득소비 · 부동산원 임대료 CSV
출력3개 시나리오 × 12개월 매출·회수기간·리스크 + PDF 1-pager (B2B 영업 무기)
CLAUDE DESIGN 적용
3개 시나리오 비교 Card 컴포넌트 + 재무 숫자의 한글 가독성(15px·행간 1.5). 결제·정산 연동 시 NaverPayButton은 공식 에셋 고정 슬롯
KPI: 데이터 일치율 ±15% · 유료 전환 의향 30%+
STEP 4 · 실행/회수

Program

Promotion → Program
이유입점 후 생존은 마케팅이 결정한다. 고객의 ③기록(리뷰·사진)을 만들어내는 촉진자 — 폐루프를 완성해 데이터가 되돌아오게 한다.
구현LLM(Claude) 프롬프트 체인으로 채널별 콘텐츠 생성 + vision 상가 이미지 분석 + 오프라인 행사 카탈로그 50종 상권 매칭
데이터네이버 검색/데이터랩(블로그·트렌드) · 카카오 로컬(현존 점포 검증) · 리뷰/이미지 크롤링 · 행사 템플릿
출력채널 믹스(인스타·블로그·당근·GFA) + LLM 콘텐츠 + A/B 테스트 + 축제·팝업 행사 매칭
CLAUDE DESIGN 적용
생성 콘텐츠의 채널별 미리보기 UI. Humanistic Authority(균형·공생·공감) 톤을 LLM 프롬프트와 UI 문구에 일관 적용
KPI: B2B 파일럿 5~10건 · 공실 해소 기간 30% 단축
4개 기능은 하나의 데이터 척추를 공유한다
